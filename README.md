## 📌 Project Overview
This project is a C++ console application that simulates a fictional “system security scenario” with password authentication and a fake system warning sequence.

The program is designed as a console interaction demo and does not perform any real system modifications.

---

## 🎯 Objectives
- Demonstrate basic C++ console programming
- Practice string handling and user input
- Use simple control flow (conditions, loops)
- Work with Windows console functions

---

## ⚙️ Technologies Used
- C++
- Windows API (`windows.h`)
- Standard libraries (`stdio.h`, `iostream`, `stdlib.h`)

---

## 🔐 Functionality
The program includes:
- Fullscreen console mode
- Fake system warning messages
- Password input system
- 3-attempt authentication mechanism
- Simulated countdown animation

---

## 🧪 How It Works
1. The program starts and requests user interaction
2. Fake system warning messages are displayed
3. The user is prompted to enter a password
4. The user has 3 attempts
5. If correct password is entered:
   - Access is granted
6. If incorrect:
   - A simulated "system deletion" countdown runs

## ⚠️ Disclaimer
This application is intended strictly for educational and demonstration purposes.

The program itself does NOT perform any automatic file deletion or system modifications.

However, since it is a console simulation, user interaction with the operating system (e.g., manually moving files or interacting with the desktop) is not controlled by the program and remains the responsibility of the user.

The “file deletion” message shown in the program is purely a simulated effect inside the console and does not represent an actual system operation initiated by the software.
